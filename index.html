Privacy Policy – Monk Mode

Last updated: 18 January 2026

Monk Mode is a productivity and focus application developed by Abhik Chakraborty. This Privacy Policy explains how information is handled when you use the app.

Information Collection and Use

Monk Mode does not collect, transmit, sell, or share any personal or sensitive user data with the developer or any third-party servers.

The app stores limited data locally on the user’s device only to enable core functionality, including:

  • Focus session state, timers, and duration
  • User-selected focus modes and preferences
  • Focus statistics and progress data

All such data remains strictly within the device’s internal storage and never leaves the device.

Internet Usage

Monk Mode may access the internet only when the user explicitly uses the in-app minimal browser for utility or study-related surfing during an active focus session.

  • No browsing history is tracked or logged by the app
  • No visited URLs are stored by the developer
  • No browsing data is transmitted externally

Permissions and Their Purpose

Accessibility Service

Accessibility permission is used solely to enforce focus restrictions, block distracting apps, detect foreground application changes, and ensure uninterrupted focus sessions. Data accessed through this permission is processed locally and never transmitted or stored externally.

Usage Access (App Usage Statistics)

Usage access is required to monitor app usage patterns during focus sessions and to provide digital wellbeing insights. All usage data remains on the device and is not shared externally.

Notification Permission (Optional)

Notification permission is used to display focus timers, session alerts, and important focus-related reminders. Notification data is not collected or transmitted.

Do Not Disturb (DND) Access

Monk Mode may request Do Not Disturb (DND) access to temporarily control notification interruptions during active focus sessions.

DND access is used only to enable or disable system-level notification suppression based on user-selected focus rules. The app does not read, collect, store, or transmit notification content, call logs, or message data when using this permission.

Make and Manage Phone Calls

Phone call permission is used to manage and handle incoming or outgoing calls during active focus sessions, such as allowing controlled call handling or enforcing focus rules. Call data is not recorded, stored, or shared.

All Files Access (Optional)

File access permission is used exclusively to allow users to open and view document files (such as PDFs or study materials) through the app’s inbuilt document viewer during focus sessions. The app accesses only files explicitly selected by the user and does not scan, monitor, index, upload, or share any files. All document processing occurs locally on the device.

Foreground Service (Special Use Permission)

Monk Mode uses a foreground service for special use cases to ensure uninterrupted operation of active focus sessions, timers, and focus enforcement features.

The foreground service runs only while a focus session is active and displays a persistent notification to inform the user that the service is running. This service does not collect, transmit, or share any personal or sensitive data and operates entirely on the device.

Battery Optimization Exemption

Battery optimization exemption is requested to ensure reliable background operation, accurate focus timers, and uninterrupted focus enforcement. This permission does not involve any data collection.

Third-Party Services

Monk Mode does not integrate third-party analytics, advertising networks, or data-collection SDKs.

Data Security

All locally stored data is protected by Android’s built-in security mechanisms. Since no personal data is transmitted externally, the risk of data exposure is minimal.
